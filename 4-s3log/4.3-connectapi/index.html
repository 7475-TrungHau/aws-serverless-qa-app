Connecting API and Logic

Connecting API and Logic

Introduction to Data Sources and Resolvers

Currently, we have an AppSync API (the entry point) and a set of Lambda functions (the processing brains), but they don’t know about each other yet. To connect them, we need to use two core concepts in AppSync:

  • Data Sources:
    This is a configuration in AppSync so it “knows” about another AWS resource, such as a Lambda function or a DynamoDB table. We will create a Data Source for each Lambda function we have created.

  • Resolvers:
    A resolver is a piece of logic attached to a specific field in the GraphQL Schema (e.g., mutation createEvent). Its job is to receive the GraphQL request, forward it to the corresponding Data Source, and then process the returned result.

In this lab, we will create the necessary Data Sources and then attach resolvers to complete the API processing flow.

Steps

1. Create a Data Source for DynamoDB

  1. Go to the AppSync Console.

  2. Select the API you are working on.

  3. Navigate to the Data Sources tab.

  4. Click Add Data Source.

  5. Name the data source: DynamoDB_Table_Source

  6. Select Amazon DynamoDB table as the Data Source type.

  7. Region: ap-southeast-1

  8. Table name: RealtimeQAPlatform-Table

  9. Click Create.

Note: This is the data source that connects directly to DynamoDB. We will use it in the resolvers later.

2. Create Data Sources for Lambda Functions

Repeat the following steps for each Lambda function you created in 4.2 Deploy Logic Functions with AWS Lambda:

  1. Click Add Data Source.

  2. Name the data source according to the corresponding Lambda function (e.g., Lambda_createEvent_Source).

  3. Select AWS Lambda Function.

  4. Region: ap-southeast-1

  5. Function: Select the corresponding function (e.g., createEventFunction)

  6. Click Create.

Note: This is the data source that connects to the Lambda Function. We will use it in the resolvers later.

List of Lambda functions to create Data Sources for:

  • createEventFunction
  • listEventsFunction
  • createQuestionFunction
  • upvoteQuestionFunction
  • createPollFunction
  • submitPollVoteFunction

3. Attach Resolvers

3.1. Attach resolver for Query getEvent(...)

  1. In the left menu, select Schema.

  2. On the right side of the page, you will see the Resolvers section. Find the Query getEvent(...): Event.

  3. Click Attach.

  4. Select Data Source: DynamoDB_Table_Source

  5. Resolver Runtime: AppSync Javascript as shown below.

  6. Click Create.

  7. In the code window, paste the following code:

    import { util } from '@aws-appsync/utils';

export function request(ctx) {
    const { id: eventId } = ctx.args;

    return {
        operation: 'Query',
        query: {
            expression: 'PK = :pk',
            expressionValues: {
                ':pk': util.dynamodb.toDynamoDB(`EVENT#${eventId}`),
            },
        },
    };
}

export function response(ctx) {
    if (ctx.error) {
        return util.error(ctx.error.message, ctx.error.type);
    }

    const items = ctx.result.items;

    if (!items || items.length === 0) {
        return util.error(`Event with id '${ctx.args.id}' not found.`, "NotFound");
    }

    let event = {};
    const questions = [];
    const polls = [];

    for (const item of items) {
        if (item.SK.startsWith('METADATA')) {
            event = item;
        } else if (item.SK.startsWith('QUESTION#')) {
            questions.push(item);
        } else if (item.SK.startsWith('POLL#')) {
            polls.push(item);
        }
    }

    event.questions = questions;
    event.polls = polls;

    return event;
}

  8. Click Save to save the resolver.

Note: getEvent() is the only resolver using AppSync Javascript. The other resolvers use Velocity Template Language (VTL).

3.2. Attach resolver for Mutation CreateEvent(...)

  1. In the left menu, select Schema.

  2. On the right side of the page, you will see the Resolvers section. Find the CreateEvent(...) function under Mutation.

  3. Click Attach.

  4. Select Data Source: Lambda_createEvent_Source

  5. Resolver Runtime: Velocity Template Language (VTL) as shown below.

  6. Click Create.

  7. After creation, a code window will appear as shown below.

Continue similarly for other Mutations/Queries with their corresponding Lambda functions:
