Giới thiệu về Data Sources và Resolvers

Hiện tại, chúng ta có một AppSync API (cổng vào) và một loạt các hàm Lambda (bộ não xử lý), nhưng chúng chưa biết gì về nhau. Để kết nối chúng, chúng ta cần sử dụng hai khái niệm cốt lõi trong AppSync:

  • Data Sources (Nguồn dữ liệu):
    Đây là một cấu hình trong AppSync để nó “biết” về một tài nguyên AWS khác, ví dụ như một hàm Lambda hoặc một bảng DynamoDB. Chúng ta sẽ tạo một Data Source cho mỗi hàm Lambda đã tạo.

  • Resolvers (Trình phân giải):
    Một resolver là một đoạn logic được gắn vào một trường (field) cụ thể trong GraphQL Schema (ví dụ: mutation createEvent). Nhiệm vụ của nó là nhận yêu cầu GraphQL, chuyển tiếp đến Data Source tương ứng, và sau đó xử lý kết quả trả về.

Trong lab này, chúng ta sẽ tạo các Data Source cần thiết và sau đó gắn các resolver để hoàn thiện luồng xử lý API.

Các bước thực hiện

1. Tạo Data Source cho DynamoDB

  1. Truy cập vào AppSync Console.

  2. Chọn API mà chúng ta đang làm việc.

  3. Di chuyển đến tab Data Sources.

  4. Click Add Data Source.

  5. Đặt tên cho data source: DynamoDB_Table_Source

  6. Chọn Amazon DynamoDB table làm loại Data Source.

  7. Chọn Region: ap-southeast-1

  8. Chọn Table name: RealtimeQAPlatform-Table

  9. Click Create.

Lưu ý: Đây là data source kết nối tới DynamoDB trực tiếp. Chúng ta sẽ sử dụng nó trong các resolver sau.

2. Tạo Data Source cho Lambda Functions

Lặp lại các bước sau cho mỗi hàm Lambda bạn đã tạo ở 4.2 Deploy các hàm Logic với AWS Lambda:

  1. Click Add Data Source.

  2. Đặt tên cho data source: Đặt tên theo hàm Lambda tương ứng (ví dụ: Lambda_createEvent_Source).

  3. Chọn AWS Lambda Function.

  4. Region: ap-southeast-1

  5. Function: Chọn Function đã tạo tương ứng (ví dụ: createEventFunction)

  6. Click Create.

Lưu ý: Đây là data source kết nối tới Lambda Function. Chúng ta sẽ sử dụng nó trong các resolver sau.

Danh sách các Lambda cần tạo Data Source:

  • createEventFunction
  • listEventsFunction
  • createQuestionFunction
  • upvoteQuestionFunction
  • createPollFunction
  • submitPollVoteFunction

3. Gắn các Resolvers

3.1. Gắn resolver cho Query getEvent(...)

  1. Trong menu bên trái, chọn Schema.

  2. Ở phía bên phải của trang, sẽ thấy mục Resolvers. Hãy tìm đến Query getEvent(...): Event.

  3. Chọn Attach.

  4. Chọn Data Source: DynamoDB_Table_Source

  5. Resolver Runtime: AppSync Javascript như hình dưới.

  6. Click Create.

  7. Trong cửa sổ code, dán đoạn code sau vào:

    import { util } from '@aws-appsync/utils';

export function request(ctx) {
    const { id: eventId } = ctx.args;

    return {
        operation: 'Query',
        query: {
            expression: 'PK = :pk',
            expressionValues: {
                ':pk': util.dynamodb.toDynamoDB(`EVENT#${eventId}`),
            },
        },
    };
}

export function response(ctx) {
    if (ctx.error) {
        return util.error(ctx.error.message, ctx.error.type);
    }

    const items = ctx.result.items;

    if (!items || items.length === 0) {
        return util.error(`Event with id '${ctx.args.id}' not found.`, "NotFound");
    }

    let event = {};
    const questions = [];
    const polls = [];

    for (const item of items) {
        if (item.SK.startsWith('METADATA')) {
            event = item;
        } else if (item.SK.startsWith('QUESTION#')) {
            questions.push(item);
        } else if (item.SK.startsWith('POLL#')) {
            polls.push(item);
        }
    }

    event.questions = questions;
    event.polls = polls;

    return event;
}

  8. Click Save để lưu resolver.

Lưu ý: getEvent() là resolver duy nhất dùng AppSync Javascript. Các hàm resolver còn lại đều dùng Velocity Template Language (VTL).

3.2. Gắn resolver cho Mutation CreateEvent(...)

  1. Trong menu bên trái, chọn Schema.

  2. Ở phía bên phải của trang, sẽ thấy mục Resolvers. Hãy tìm đến hàm CreateEvent(...) ở Mutation.

  3. Chọn Attach.

  4. Chọn Data Source: Lambda_createEvent_Source

  5. Resolver Runtime: Velocity Template Language (VTL) như hình dưới.

  6. Click Create.

  7. Sau khi tạo xong sẽ hiện ra cửa sổ code như hình dưới.

Tiếp tục thực hiện tương tự cho các Mutation/Query khác với các Lambda tương ứng:

Tên Mutation/Query Data Source Resolver Runtime
CreateQuestion(...) Lambda_createQuestion_Source Velocity Template Language (VTL)
UpvoteQuestion(...) Lambda_upvoteQuestion_Source Velocity Template Language (VTL)
SubmitPollVote(...) Lambda_submitPollVote_Source Velocity Template Language (VTL)
CreatePoll(...) Lambda_createPoll_Source Velocity Template Language (VTL)
listEvents Lambda_listEvents_Source Velocity Template Language (VTL)

Lưu ý:

  • Đảm bảo chọn đúng Data Source tương ứng với từng Lambda Function.
  • Tất cả các resolver này đều sử dụng Velocity Template Language (VTL).
  • Thực hiện các bước tương tự như phần hướng dẫn cho CreateEvent(...).
